{{< img-text title="Code & Architektur" src="/images/Coding.png" >}}
Dieses Projekt konzentriert sich auf den Aufbau eines persönlichen Portfolios von Grund auf, wobei bewusst auf „Drag-and-Drop“-Baukästen verzichtet wurde, um die volle Kontrolle über den zugrunde liegenden Quellcode zu gewährleisten. Entwickelt in Visual Studio Code, basiert die Seite auf Hugo, einem schnellen und flexiblen Generator für statische Seiten (SSG). Für die Frontend-Architektur habe ich HTML in Kombination mit Tailwind CSS verwendet, dessen „Utility-First“-Ansatz ein effizientes Styling und eine saubere, responsive Codebasis ermöglicht.
{{< /img-text >}}

{{< img-text title="Design-Philosophie" src="/images/Bento-grid.png" reverse="true" >}}
Die visuelle Identität ist in einer Dark-Mode-Ästhetik verankert, die modern und elegant wirkt und gleichzeitig die Augenbelastung reduziert. Um eine klare visuelle Hierarchie zu schaffen, habe ich kontrastreiche Neon-Akzente integriert, die den Fokus auf Schlüsselelemente lenken. Der Inhalt ist in einem Bento-Grid-Layout organisiert, das Informationen in modulare, interaktive Kacheln unterteilt. Zur Optimierung der User Experience (UX) habe ich flüssiges Motion Design implementiert – einschließlich weicher Übergänge und Mikro-Interaktionen –, was eine nahtlose und intuitive Navigation gewährleistet.
{{< /img-text >}}

{{< img-text title="Deployment" src="/images/GitHub.png" >}}
Die finale Phase umfasste das Deployment der statischen Seite via GitHub Pages. Dieser Workflow integriert git für eine rigorose Versionskontrolle, wodurch ich die Entwicklungshistorie nachverfolgen und Branches (Entwicklungszweige) effizient verwalten kann. Dieses Setup beschleunigt den Iterationszyklus erheblich und ermöglicht es mir, Fehler schnell zu identifizieren (Debugging) und neue Inhalte oder Updates unmittelbar auf den Live-Server zu pushen.
{{< /img-text >}}
